MCFast Comments on Spurious Error Messages

August 24, 1998

There are three sorts of error messages where are generated by the example and which appear to be ignorable. They currently being investigated.

(A) On an IRIX64 machines with the -n32 compilation switch many error messages are generated at link-time. There are several complaints about libraries not being used to resolve any symbols. In addition there are the following messages:

ld32: WARNING 16: slate_ used in /usr/products/IRIX6/cern/v97aa/lib/libpacklib.a(datime.o)
and /usr/products/IRIX6/mcfast/v4_0/lib.IRIX64/libdbin.a(dbinf.o)
has different sizes.
See explanation in the gp_overflow(5) manpage.

ld32: WARNING 16: slate_ used in /usr/products/IRIX6/cern/v97aa/lib/libpacklib.a(icnext.o) and /usr/products/IRIX6/cern/v97aa/lib/libpacklib.a(datime.o) has different sizes. See explanation in the gp_overflow(5) manpage.

ld32: WARNING 85: definition of invalidop_operands_ in /usr/products/IRIX6/mcfast/v4_0/lib.IRIX64/libgeneral.a(fp_exception.o) preempts that definition in /usr/lib32/mips3/libfpe.so.

ld32: WARNING 85: definition of invalidop_results_ in /usr/products/IRIX6/mcfast/v4_0/lib.IRIX64/libgeneral.a(fp_exception.o) preempts that definition in /usr/lib32/mips3/libfpe.so.

ld32: WARNING 85: definition of sigfpe_ in /usr/products/IRIX6/mcfast/v4_0/lib.IRIX64/libgeneral.a(fp_exception.o) preempts that definition in /usr/lib32/mips3/libfpe.so.

ld32: WARNING 85: definition of main in /usr/products/IRIX6/mcfast/v4_0/lib.IRIX64/main_args.o preempts that definition in /usr/lib32/mips3/libftn.so.

ld32: WARNING 84: /usr/lib32/libsun.a is not used for resolving any symbol.

ld32: WARNING 85: definition of __checktraps in /usr/lib32/mips3/libfpe.so preempts that definition in /usr/lib32/mips3/libc.so.

ld32: WARNING 85: definition of __readenv_sigfpe in /usr/lib32/mips3/libfpe.so preempts that definition in /usr/lib32/mips3/libc.so.

(B) On an AIX machine the following link-time messages are generated:

ld: 0711-224 WARNING: Duplicate symbol: qcdpar_
ld: 0711-224 WARNING: Duplicate symbol: signgam
ld: 0711-341 WARNING: Replaced XTY_CM symbol signgam requires an alignment more strict than the alignment of the symbol that replaces it.

(C) Whenever events are written using the mcfio_out command. The folowing messages appear. This happens on all platforms.

mcfio_GetBlockName: Unrecognized block I.D. 880
mcfio_GetBlockName: Unrecognized block I.D. 881
mcfio_GetBlockName: Unrecognized block I.D. 701
